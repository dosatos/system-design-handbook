## Key DP Patterns

### 1D DP Patterns
1. **Fibonacci-style**: f(n) = f(n-1) + f(n-2)
2. **Min/Max to reach**: dp[i] = min/max of choices
3. **Decision at each step**: take or skip
4. **Partition problems**: can we achieve target?

### 2D DP Patterns
1. **Grid paths**: dp[i][j] = paths to reach (i,j)
2. **Two strings**: dp[i][j] = result for str1[:i] and str2[:j]
3. **Interval DP**: dp[i][j] = result for subarray [i,j]
4. **Knapsack variants**: dp[i][w] = value with i items, capacity w

### DP Approach
1. Define state clearly
2. Write recurrence relation
3. Identify base cases
4. Decide iteration order
5. Optimize space if possible
