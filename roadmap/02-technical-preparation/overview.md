## Daily Time Allocation

**Recommended: 3-4 hours/day**

| Activity | Time | Frequency |
|----------|------|-----------|
| Coding Practice | 60-90 min | Daily |
| System Design Study | 45-60 min | Daily |
| ML/AI Concepts | 30-45 min | Daily |
| Mock Interviews | 60-90 min | 2-3x/week |
| Project Work | 2-3 hours | Weekends |

## Weekly Rhythm

### Weekdays (3-4 hours)

```
Morning (1.5 hours):
├── Coding warm-up: 1 easy problem (15 min)
├── Coding practice: 1-2 medium problems (45 min)
└── Review solutions and patterns (30 min)

Evening (1.5-2 hours):
├── System Design: Read/practice (45-60 min)
├── ML/AI concepts: Study/implement (30-45 min)
└── Review/flashcards (15-30 min)
```

### Weekends (4-6 hours)

```
Saturday:
├── Mock interview or hard problems (2 hours)
├── System design deep dive (1.5 hours)
└── Project work (1.5 hours)

Sunday:
├── Week review and planning (1 hour)
├── Weak area focus (2 hours)
└── Project work (2 hours)
```

## Language Recommendations

**Primary: Python**
- All AI companies strongly prefer Python
- Fastest to write in interviews
- Best for ML-related questions

**Secondary (know basics):**
- C++ (DeepMind, performance-critical roles)
- Go/Rust (xAI, infrastructure roles)
- JavaScript (full-stack roles)

## Common Pitfalls to Avoid

1. **Grinding without understanding** - Learn patterns, not memorize solutions
2. **Ignoring system design** - Critical for senior roles
3. **Skipping ML fundamentals** - AI companies test this
4. **Neglecting behavioral** - Can be a dealbreaker
5. **Not doing mock interviews** - Essential for real performance
6. **Cramming before interviews** - Spread preparation over weeks

## Adjusting the Schedule

### If you have less time (6-8 weeks):
- Focus on Blind 75 only for coding
- Prioritize ML system design over classical
- Do 1-2 mock interviews per week
- Skip some project work

### If you have more time (12-16 weeks):
- Complete NeetCode 150
- Add more mock interviews
- Build more substantial projects
- Deep dive into company-specific prep
