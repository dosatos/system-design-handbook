# Resume Optimization for AI Companies

## Key Principles

### 1. Lead with Impact, Not Tasks

**Bad:**
```
- Worked on ML infrastructure
- Helped improve model performance
- Participated in code reviews
```

**Good:**
```
- Reduced model inference latency by 40% by implementing dynamic batching,
  serving 10M+ daily requests
- Led migration from TensorFlow to PyTorch, improving training iteration
  time by 60% across 5 teams
- Designed fault-tolerant distributed training system supporting 100B+
  parameter models
```

### 2. Use Company-Relevant Keywords

**For AI Companies:**
- LLMs, transformers, attention mechanisms
- Distributed training, model parallelism
- Inference optimization, model serving
- AI safety, interpretability, alignment
- RLHF, Constitutional AI
- RAG, embeddings, vector databases
- Production ML, MLOps

### 3. Highlight Independent Work

From Anthropic:
> "If you've done interesting independent research, written a thoughtful blog post, or contributed to open source, put that at the TOP of your resume."

**What to Highlight:**
- Open source contributions
- Technical blog posts
- Side projects
- Research publications
- Speaking engagements

## Section-by-Section Guide

### Summary

**Template:**
```
[Role] with [X years] experience in [key areas].
[Major achievement or specialization].
[What you're looking for / what you bring].
```

**Example:**
```
Senior Software Engineer with 7 years experience building ML systems
at scale. Led development of inference infrastructure serving 50M+
daily predictions at <50ms latency. Deep expertise in LLM deployment,
distributed training, and production ML systems.
```

### Notable Projects (For AI Companies, put this before Experience!)

**Format:**
```
[Project Name] | [Link]
• What it does (1 line)
• Key metrics or impact
• Technologies used
```

**Example:**
```
Distributed Training Framework | github.com/you/dist-trainer
• PyTorch training framework for multi-GPU scaling
• 40% training speedup vs baseline through gradient compression
• Used by 3 internal teams for production model training
```

### Experience

**Use this formula:**
```
[Action verb] + [What you did] + [Scale/Context] + [Result/Impact]
```

**Strong Verbs for AI Roles:**
- Designed, architected, built
- Led, owned, drove
- Optimized, reduced, improved
- Scaled, migrated, deployed
- Researched, evaluated, implemented

**Examples:**
```
• Designed distributed training system for 70B parameter models,
  reducing training time by 3x through optimized gradient synchronization

• Led migration from batch to real-time ML inference, reducing latency
  from 500ms to 50ms while maintaining 99.9% availability

• Built RAG pipeline processing 10M documents with hybrid search,
  achieving 0.85 Recall@10 for enterprise knowledge base
```

### Skills

**Structure by Category:**
```
Languages: Python (primary), C++, Go, Rust
ML Frameworks: PyTorch, TensorFlow, JAX, Hugging Face
Infrastructure: Kubernetes, Docker, AWS (EC2, S3, SageMaker)
Data: PostgreSQL, Redis, Kafka, Apache Spark
ML Tools: Weights & Biases, MLflow, Ray, vLLM
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Too long (3+ pages) | 1-2 pages maximum |
| Generic bullets | Specific metrics and impact |
| Buzzword stuffing | Genuine, detailed experience |
| No links | Include GitHub, blog, portfolio |
| Outdated format | Clean, modern, ATS-friendly |
| Missing keywords | Match job description |
| Tasks instead of achievements | Focus on impact |

---

## ATS Optimization

**Do:**
- Use standard section headers
- Include keywords from job posting
- Use simple formatting (avoid tables, columns)
- Save as PDF and .docx

**Don't:**
- Use images or graphics
- Use unusual fonts
- Use headers/footers
- Use complex layouts

---

## Resume Checklist

### Content
- [ ] Summary is specific and compelling
- [ ] Notable projects highlighted (for AI companies)
- [ ] All bullets show impact with metrics
- [ ] Keywords match target job descriptions
- [ ] Links to GitHub, portfolio, blog

### Format
- [ ] 1-2 pages maximum
- [ ] Clean, consistent formatting
- [ ] Easy to scan
- [ ] No typos or grammatical errors

### Customization
- [ ] Tailored for target company
- [ ] Relevant experience emphasized
- [ ] Skills section matches job requirements

---

## Final Tips

1. **Get feedback** - Have others review
2. **Quantify everything** - Numbers stand out
3. **Update regularly** - Don't wait until job hunting
4. **Keep a master resume** - Full version to customize from
5. **Match the JD** - Mirror language from job postings
