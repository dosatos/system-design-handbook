## Interview Tips Specific to xAI

1. **Expect LeetCode Hard** - Higher difficulty than other AI labs
2. **Production-ready code** - Write clean, concurrent code live
3. **First-principles thinking** - Derive solutions from fundamentals
4. **Speed matters** - Demonstrate velocity in problem-solving
5. **Show initiative** - Stories about taking ownership without being asked
6. **Distributed systems depth** - Know sharding, consistency, latency trade-offs

## Sample Interview Questions

### Coding (Hard difficulty)
- **Alphabet Word Search (Boggle):** Given NxN board and dictionary, find all words
  - Solution: Trie + DFS backtracking
- **LRU Cache:** get(key) and put(key, value) with O(1) complexity
  - Solution: Hash table + bidirectional linked list
- **Concurrent Data Structure:** Multiple reader/writer threads with minimal lock contention
  - Solution: Sharded maps, read-write locks

### System Design
- "Design a high-throughput log ingestion pipeline"
- "Design model serving for real-time reasoning with multi-version A/B testing"

**Focus areas:** Scalability, low latency, high reliability, data consistency

### ML Questions (if applicable)
- Trade-offs between data parallelism and model parallelism
- Implement simplified Transformer component
- Write an efficient tokenizer
