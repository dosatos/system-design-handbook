# User Preferences & Learning Style

## Communication Preferences

### Tone
- Direct and efficient communication
- Skip unnecessary pleasantries in work sessions
- Be encouraging but not patronizing
- Challenge me when I'm taking shortcuts

### Feedback Style
- Give honest, constructive feedback
- Don't sugarcoat areas that need improvement
- Celebrate genuine progress, not participation
- Provide specific, actionable suggestions

### Explanation Depth
- Start with high-level concepts, then dive deep on request
- Use analogies to complex systems I already understand
- Provide code examples in Python (primary) or TypeScript (secondary)
- Link concepts to real-world AI lab scenarios

## Learning Style

### Problem Solving Approach
- Prefer Socratic method for coding problems (hints before solutions)
- Want to understand the "why" behind patterns, not just memorize
- Appreciate seeing multiple approaches with trade-off analysis
- Learn best by doing, then reviewing

### Session Structure
- Prefer focused 1-2 hour deep work sessions
- Appreciate clear session goals upfront
- Want progress tracking at end of sessions
- Like having "homework" or async tasks to complete

### Knowledge Gaps to Address
- Distributed systems (CAP theorem, consensus algorithms)
- ML infrastructure at scale (training pipelines, serving)
- Advanced Python performance optimization
- Behavioral storytelling (STAR format)

## Technical Preferences

### Languages & Tools
- Primary: Python, SQL
- Secondary: TypeScript, Go
- Familiar: Java, C++
- Tools: Git, Docker, Kubernetes, AWS/GCP

### Code Style
- Prefer clean, readable code over clever one-liners
- Type hints in Python
- Comprehensive docstrings for complex functions
- Test-driven development when appropriate

### System Design Approach
- Start with requirements clarification
- Draw high-level architecture first
- Dive into specific components as needed
- Always discuss trade-offs and alternatives

## Schedule & Availability

### Typical Prep Schedule
- Weekday mornings: 1-2 hours before work
- Weekday evenings: 2-3 hours focused prep
- Weekends: 4-6 hours split between coding and study
- Target: 20-25 hours/week dedicated prep

### Peak Focus Times
- Best for hard problems: Morning (8-11 AM)
- Best for review/reading: Evening (8-10 PM)
- Least productive: Early afternoon (2-4 PM)

## Motivations & Goals

### Why These Companies
- Work on cutting-edge AI that impacts millions
- Collaborate with world-class researchers and engineers
- Compensation that provides financial security
- Be part of shaping AI safety and alignment

### Career Vision
- Short-term: Senior SWE at top AI lab
- Medium-term: Tech Lead / Staff Engineer
- Long-term: Technical leadership in AI safety

### Non-Negotiables
- Must be technically challenging work
- Must have impact on real products/research
- Must have strong engineering culture
- Remote flexibility preferred but not required for right role

## Stress & Motivation Triggers

### What Motivates Me
- Seeing measurable progress (problems solved, topics mastered)
- Understanding complex systems deeply
- Positive mock interview feedback
- Networking wins and referral opportunities

### What Causes Stress
- Feeling unprepared or behind schedule
- Struggling with same problem type repeatedly
- Rejection without clear feedback
- Unclear expectations in interviews

### How to Help When Stuck
- Break problem into smaller parts
- Remind me of similar problems I've solved
- Suggest a 10-minute break, then fresh approach
- Provide one targeted hint, then let me try again
